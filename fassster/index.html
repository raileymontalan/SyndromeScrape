What is FASSSTER?

 

Feasibility Analysis of Syndromic Surveillance Using Spatio-Temporal Epidemiological Modeler For Early Detection of Diseases

FASSSTER aims to create a hub for different data sources (Disease records, environmental parameters, etc.), providing a rich layout of integrated information that will facilitate in understanding the spread of diseases.

In partnership with the Department of Health, Department of Science and Technology - Philippine Council for Health Research and Development, and the Ateneo de Manila University, FASSSTER aims to provide a user-friendly tool for modeling disease spreads in the Philippines to aid in the DOH's disease surveillance efforts. FASSSTER takes a multi-dimensional approach in modeling disease spreads by using localized indices from Philippine health records and by integrating third-party data, such as those found in Electronic Medical Record (EMR) systems and online sources.

Disease Modeling

All disease models are made using IBM's Spatio-Temporal Epidemiological Modeler (STEM).

Disease Models:

  1. Dengue Fever Disease Model
  2. Measles Disease Model
  3. Typhoid Fever Disease Model

It follows the compartmental model approach, where a person is classified under a disease state compartment (e.g. Susceptible, Infectious, Recovered, etc.) at a given point in time. A scenario in STEM is built by combining different factors such as the initializers, schedulers, population and geographic models, and more.

Integrating FASSSTER, indices used in the disease models come from official records such as the Department of Health's Philippine Integrated Disease Surveillance and Response (PIDSR) System and Philippine Health Statistics.

For Project FASSSTER, Disease Modelers work on building the initial models using the standalone STEM Application. Once done, the model is uploaded and viewed via the Web Application.

Web Application

The web application is designed to allow epidemiologist to modify disease models in a simpler and more user-friendly interface.

All technologies used are open source, including the Python-Tornado web framework, MongoDB NoSQL database, and LeafletJS.

Based on the uploaded disease models, the Web Application triggers the standalone STEM application located in the server, and visualizes the results on a map. The Simulation Managers can then modify the different parameters directly from the Web Application to create their own "what if" scenarios.

Features:

  1. Open source technologies utilization (Python, MongoDB, LeafletJS)
  2. Standalone STEM Application headless trigger
  3. Disease spread prediction visualization

Syndromic Surveillance Visualization

For its second year, the project aims to integrate third-party sources into the disease models themselves. Through the use of Application Programming Interfaces (APIs), the Web Application becomes a hub for disease modelers to include live deidentified data from significant surveillance systems. Since APIs are used, no data is stored in FASSSTER's server.

Integrated data sources are:
  • SHINE OS+
  • Infodemiological (Health-Related) Tweets
  • PIDSR Data
